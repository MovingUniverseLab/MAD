Welcome to the Microlensing Alert Database (MAD)!

Use SQL query syntax to query the microlensing.db database.
Tables in database available to query are "alerts" and "photometry".

Default SQL query: SELECT * FROM alerts WHERE ABS(b)<10 AND tE>=100 AND t0>=({{current_mjd}} - 14) AND Ibase<20 ORDER BY t0_err, tE_err ASC

The columns for the "alerts" table are listed in the table below.
Note that KMTNet does not provide uncertainties on the parameter estimates.

Column name Description
alert_name e.g. MBYYNNN, OBYYNNNN, KBYYNNNN
RA J2000.0
Dec J2000.0
l Galactic longitude
b Galactic latitude
t0, t0_err Time of maximum brightness (1)
tE, tE_err Einstein crossing time (2)
u0, u0_err Dimensionless impact parameter (3)
Ibase, Ibase_err I-band baseline magnitude (4)
Isrc, Isrc_err I-band baseline magnitude (5)
srcfrac, srcfrac_err Source flux fraction (6)
class Event classification (7)
alert_url URL to event alert page
related_event Event name if the event is recorded by another telescope

(1) This is named "tmax" on MOA and OGLE alert sites.
(2) This is named "tau" on the OGLE alert site.
(3) This is named "Umin" on the OGLE alert site.
(4) This is named "Ibl" on the OGLE alert site.
(5) This is named "I0" on the OGLE alert site. MOA does not report this parameter.
(6) This is named "fbl" on the OGLE alert site. MOA does not report this parameter.
KMTNet does not report this parameter, but we calculate it from Ibase and Isrc.
(7) This is named "assessment" on the MOA alert site, with values like "microlensing" or "cv".
OGLE only reports microlensing events, so we have labelled all OGLE events as "microlensing".
KMTNet classifies events by their probability of being microlensing events, such as "clear" or "probable".
For years where there are two classifications (EF and AL, I cannot find what the meaning of these two are),
we arbitrarily have reported the EF classification.
